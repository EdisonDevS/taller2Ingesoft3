Este caso de uso ocurre cuando un nuevo suscriptor desea pagar su tarifa de suscripción anual especificando un número de tarjeta de crédito y un PIN. Esto también puede ocurrir cuando un suscriptor existente desea renovar (ver flujo alternativo 1)<br/><br/>

Vista de pasarela de pago Functional Proposed Medium Medium Moderate
Details:
Es necesario verificar las especificaciones de la interfaz para el sistema de tarjeta de crédito externo<br/>
Scenario Type
Flujo básico Basic Path
Details:
Flujo normal del caso de uso<br/>
Steps:
1. El suscriptor selecciona "pagar tarifa con tarjeta de crédito"<br/>
2. El sistema solicita al suscriptor el número de la tarjeta de crédito, la fecha de vencimiento y (opcionalmente) el PIN<br/>
3. El sistema envía la información de la tarjeta de crédito al sistema externo para la validación y aplicación del cargo<br/>
4. Al recibir la validación, el sistema actualiza el registro del suscriptor para indicar una nueva fecha de vencimiento<br/>
El suscriptor renueva la suscripción Alternate
Details:
Cuando esto ocurre, el flujo se ejecuta de la siguiente manera<br/>
Steps:
1. El suscriptor selecciona "pagar tarifa con tarjeta de crédito"<br/>
2. El sistema muestra la información actual de la tarjeta de crédito<br/>
3. El usuario acepta la información tal como está o la actualiza adecuadamente<br/>
4. El sistema envía la información de la tarjeta de crédito al sistema externo para la validación y aplicación del cargo<br/>
5. Al recibir la validación, el sistema actualiza el registro del suscriptor para indicar una nueva fecha de vencimiento<br/>
Información de tarjeta de crédito no válida Alternate
Details:
Si la información proporcionada por el suscriptor no es validada por el sistema externo, se mostrará un mensaje de error y NO se actualizará el registro del suscriptor (por lo que no se ejecutarán los últimos pasos en los flujos anteriores).<br/>

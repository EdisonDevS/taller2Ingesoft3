Este caso de uso ocurre cuando un anunciante desea publicar contenido publicitario (anuncios publicitarios) en el sitio web y especificar qué perfiles de suscriptor se deben usar para mostrar<br/>

Constraint Type Status
Existe una cuenta de anunciante Invariant Approved
Details:
La cuenta de anunciante se encuentra registrada en la base de datos<br/>
El usuario está conectado y validado como anunciante Invariant Approved
Details:
El usuario pasó por el proceso de autenticación para su rol correctamente<br/>
Scenario Type
Flujo básico Basic Path
Details:
Flujo normal del caso de uso<br/>
Steps:
1. El anunciante selecciona "Publicar contenido"<br/>
2. El sistema valida la información de facturación de la cuenta para garantizar que se acepte contenido nuevo<br/>
3. Mensajes del sistema para el contenido<br/>
4. El anunciante sube contenido en formato GIF<br/>
5. El sistema muestra categorías potenciales para la visualización de anuncios (según las opciones del perfil del suscriptor)<br/>
6. El anunciante selecciona las categorías para las que se debe mostrar este anuncio<br/>
7. El sistema muestra las frecuencias y los precios potenciales del anuncio<br/>
8. El anunciante selecciona la frecuencia deseada para este anuncio<br/>
9. El sistema crea un registro de facturación preliminar para este anuncio<br/>
10. El sistema coloca el contenido en el flujo de trabajo de "tareas pendientes" del editor para su aprobación<br/>
Información de cuenta no válida Alternate
Details:
Ocurre algún error en la facturación<br/>
Steps:
1. El anunciante selecciona "Publicar contenido"<br/>
2. El sistema valida la información de facturación de la cuenta para garantizar que se acepte contenido nuevo<br/>
3. La información de la cuenta no es válida, se solicita al anunciante que se comunique con el departamento de facturación<br/>

Este caso de uso ocurre cuando el contenido se publica en el sitio web <b><i>WebNewsOnLine</i></b> existente. Algunas historias se etiquetarán para su transmisión al sistema de localización de deportes universitarios y se enviarán para su posible localización y visualización.<br/><br/>

Constraint Type Status
El editor debe estar conectado Invariant Approved
Details:
El usuario debe haber realizado el proceso de autenticación de editor correctamente<br/>
Scenario Type
Flujo básico Basic Path
Details:
Flujo normal del caso de uso<br/>
Steps:
1. El caso de uso comienza cuando un editor de contenido coloca contenido en el sitio web WebNewsOnLine.<br/>
2. El sistema WebNewsOnLine verifica la categorización y / o el título del contenido recién publicado.<br/>
3. El sistema verifica la información de referencia para determinar si esta categoría de contenido es conocida por el sistema de búsqueda de deportes universitarios.<br/>
4. Para las categorías que son conocidas por el sistema colegiado de avisos deportivos, las historias se transmiten al sistema colegiado de avisos deportivos junto con la información de la categoría.<br/>
5. El contenido se coloca en el almacenamiento Sistema de Avisos Deportivos Universitarios para una posible paginación.<br/>
El contenido no está categorizado Alternate
Details:
Si el contenido no está categorizado, la historia se coloca en el flujo de trabajo de "tareas pendientes" del editor.<br/><br/>
